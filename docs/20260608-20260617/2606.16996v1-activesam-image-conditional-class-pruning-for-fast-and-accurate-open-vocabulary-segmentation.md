## 摘要
Segment Anything Model 3 (SAM 3) 为概念驱动的分割提供了强大的冻结骨干网络，但直接将其应用于开放词汇语义分割（OVSS）效率低下：通常会在整个数据集词汇表上运行全分辨率解码，而每张图像仅包含一小部分活跃的类别子集。我们提出了 ActiveSAM，这是一种无需训练、零样本推理框架，可将 SAM 3 转变为主动词汇分割器。ActiveSAM 首先对类别提示进行规范化并扩展，然后通过低分辨率存在性预览估计图像条件下的活跃类别集。仅保留的类别会以全分辨率解码，使用与冻结的 SAM 3 解码器配合的分桶提示复用技术。预览阶段仅利用类别存在证据，跳过不必要的分割头计算，而最终阶段则采用边缘感知的背景校准来抑制低置信度像素。ActiveSAM 不需要目标数据集训练、无需更新权重，也无需确实的类别存在标签。在八个 OVSS 基准测试中，ActiveSAM 改善了无需训练的开放词汇语义分割的速度-精度权衡，在大型词汇数据集上，平均 mIoU 比当前最优的 SegEarth-OV3 提升约 +1.4，同时运行速度提升了 5.5 倍。ActiveSAM 在模拟真实世界分布偏移的图像损坏下也表现出最强的鲁棒性，使其非常适合部署在噪声输入领域，如自动驾驶和具身人工智能。代码可在 https://github.com/VILA-Lab/ActiveSAM 获取。

## Abstract
Segment Anything Model 3 (SAM 3) provides a strong frozen backbone for concept-prompted segmentation, but applying it directly to open-vocabulary semantic segmentation (OVSS) is inefficient: full-resolution decoding is typically run over the entire dataset vocabulary, whereas each image contains only a small active subset of classes. We introduce ActiveSAM, a training-free, zero-shot inference framework that turns SAM 3 into an active-vocabulary segmenter. ActiveSAM first canonicalizes and expands class prompts, then estimates an image-conditioned active set from a low-resolution presence preview. Only the retained classes are decoded at full resolution, using bucketed prompt multiplexing with the frozen SAM 3 decoder. The preview stage uses only class-presence evidence and skips unnecessary segmentation-head computation, while the final stage applies margin-aware background calibration to suppress low-confidence pixels. ActiveSAM requires no target-dataset training, no weight updates, and no oracle class-presence labels. Across eight OVSS benchmarks, ActiveSAM improves the speed-accuracy tradeoff of training-free open-vocabulary semantic segmentation, outperforming the current state-of-the-art SegEarth-OV3 by approximately +1.4 mIoU on average while running up to 5.5x faster on large-vocabulary datasets. ActiveSAM also demonstrates the strongest robustness under image corruption that simulates real-world distribution shift, making it well-suited for deployment in noisy-input domains such as autonomous driving and embodied AI. Code is available at https://github.com/VILA-Lab/ActiveSAM.

## 1. 论文的核心问题与整体含义

- **核心问题**：在开放词汇语义分割（OVSS）中，直接使用分割基础模型 SAM 3 需要为整个数据集的词汇表（可能包含上百个类别）逐一执行全分辨率提示解码。然而，单张图像通常只包含其中极少数的“活跃”类别，导致大量计算浪费在无效的类别解码上。
- **整体含义**：通过引入**图像条件的类别剪枝**机制，在推理时动态筛选出每张图像可能出现的少数类别，只对它们进行高成本的全分辨率解码，从而在保持甚至提升分割精度的前提下，大幅降低计算开销。该框架无需任何训练，可零样本部署，为开放词汇分割的实际应用（如自动驾驶、具身智能）提供了高效、鲁棒的解决方案。

## 2. 方法论

整个框架**保持 SAM 3 全部权重冻结**，由四个关键阶段组成：

- **上下文提示扩展 (Contextual Prompt Expansion, CPE)**
  - **目的**：增强原始类别名称的语义丰富度，弥补孤立的文本提示在视觉对齐上的不足。
  - **做法**：
    1. **词汇规范化**：通过预定义的字符串映射函数 `η` 修复标签中的拼接或倒序问题（如 `windowpane` → `window pane`）。
    2. **邻近类别记忆**：在数据集词汇内，根据冻结的文本嵌入余弦相似度，为每个类别检索 `M_s` 个最邻近的类别名称，编码后作为额外的提示令牌。
    3. **WordNet 超义词记忆**：查询每个类别在 WordNet 中的前 `M_t` 个名词超义词，同样编码为提示令牌。
    4. 将原始类名令牌、邻近类别令牌、超义词令牌拼接，构成每类的最终上下文提示 `Z̄_c`，该提示在数据集层面**预计算并缓存**，图像推理时直接复用。

- **预览驱动的类别选择 (Preview-driven Class Selection)**
  - **目的**：以极低的成本估计图像中哪些类别可能存在，从而剪枝掉无关类别。
  - **做法**：
    1. 使用冻结的 SAM 3 在**低分辨率**（672×672）下执行一次前传，**仅提取类别存在头（presence head）的 logit**，跳过分割掩码的输出。
    2. 对存在分数 `q_c` 应用图像自适应的分位数阈值 `τ(I)`（默认 `β=0.80`），筛选出候选活跃集。
    3. 引入门控阈值 `V_gate=40`：若候选集大小超过 `V_gate`，则只保留分数最高的 `V_gate` 个类别；若数据集总类别数 `V ≤ V_gate`，则直接跳过预览，处理全部词汇。
  - **效果**：将每张图像实际解码的类别数平均缩减至 6–30 个（原始词汇达 59–171 类）。

- **分桶全分辨率解码 (Bucketed Full-Resolution Decoding)**
  - **计算图像特征一次**：在**全分辨率**（1008×1008）下运行冻结的 SAM 3 图像编码器，得到共享特征 `X`。
  - **分桶提示复用**：将筛选出的活跃类别 `A(I)` 按每桶最多 `K=32` 个类别分组。对于每一桶，将其所有类别的上下文提示拼接后，一起送入冻结的 SAM 3 解码器。
  - 融合每个类别的实例头分数 `S_inst` 和语义头分数 `S_sem`（按像素取最大），再乘以全分辨率下的存在分数，得到最终的类别分数图 `S_c(p)`。

- **边缘感知背景校准 (Margin-Aware Background Calibration, MABC)**
  - **目的**：对于包含背景类的数据集，用更严格的决策边界替代简单的分数阈值，减少低置信度误检。
  - **做法**：
    1. 对每个像素，取最高分 `s1` 和次高分 `s2`，计算校准置信度 `c_MABC = s1 * max(s1 - s2, 0)`。
    2. 若 `c_MABC < t^γ`（`t` 为数据集原始背景阈值，`γ=1.25`），则将该像素标记为背景；否则保留预测的最高分类别。无背景类的数据集跳过此步。

## 3. 实验设计

- **数据集与基准**：在8个标准开放词汇语义分割基准上评测：

  - 不含背景类的：Pascal VOC 2020 (`V20`), Pascal Context 59 (`P59`), COCO-Stuff (`C-S`), Cityscapes (`City`), ADE20K (`ADE`)。
- **对比方法**：与**训练式**（如 GroupViT, SegCLIP, Talk2DINO）和**训练自由式**（如 CLIP, SCLIP, ProxyCLIP, CorrCLIP, RF‑CLIP, SAM 3原版, SegEarth‑OV3）的十余种 SOTA 方法进行对比。
- **评估指标**：报告各类别的平均交并比 (mIoU)，以及每张图像的推理帧率 (FPS) 进行速度对比。
- **鲁棒性测试**：在 ImageNet‑C 的四种代表性损坏（高斯噪声、运动模糊、JPEG压缩、雾）下，对比 ActiveSAM、SegEarth‑OV3、CorrCLIP、RF‑CLIP 的性能下降程度。
- **组件消融**：通过逐一移除 CPE、MABC、类别剪枝来验证其贡献。

## 4. 资源与算力

- **硬件条件**：所有实验均在**单张 NVIDIA RTX 5090** 上完成。

## 6. 论文的主要结论与发现

- ActiveSAM 在 8 个基准上取得 **65.3% 的平均 mIoU**，比当时最强的 SAM 3‑based 方法 SegEarth‑OV3 高出 **+1.4 mIoU**，比原始 SAM 3 高出 **+7.8 mIoU**。
- 在大型词汇数据集（如 ADE20K, COCO‑Stuff）上，推理速度可达 SegEarth‑OV3 的 **4.8–5.5 倍**，同时精度更高，证实了全词汇解码存在大量冗余。
- 类别剪枝（预览→筛选）带来了显著的速度增益，但 **对最终分割精度影响为零**（消融实验）。
- 在图像损坏（噪声、模糊、压缩、雾）条件下，ActiveSAM 的性能衰减远小于依赖 CLIP 特征的方法，展现出**极强的鲁棒性**，非常适合真实世界的感知系统。
- CPE 和 MABC 两个质量模块对提升分割精度有实质贡献。

## 7. 优点

- **高效的计算范式转换**：将“先全量解码、后筛选”的常规流程转变为“先低成本预览、后针对性解码”，巧妙利用 SAM 3 的内置存在头，计算力分配极具针对性。
- **完全即插即用**：无需任何训练、微调或特定数据集的标注，模型权重全程冻结，可随时切换部署词汇。
- **显著的速度精度双赢**：不仅大幅提速，精度更是全面超越原有方法，尤其在大型词汇表上收益巨大。
- **优异的鲁棒性**：不依赖对图像扰动敏感的逐块视觉‑文本对齐，天然具备对真实世界噪声的抵抗力。
- **方法简洁，可解释性强**：每个模块设计动机清晰（语义丰富、计算剪枝、背景决策），消融验证完整，代码开源。

## 8. 不足与局限

- **对极小或遮挡目标的召回风险**：低分辨率预览的存在头如果漏掉极小的物体，该类别将被永久排除在最终解码之外，没有补救机会。这是“剪枝”机制固有的取舍。
- **词汇扩展依赖英文词库**：CPE 依赖 WordNet 和人工编写的英文规范化规则，对于**非英语概念、新兴网络词、高度专业领域术语**（如特定医疗、工业术语）的泛化能力可能受限。
- **基准覆盖的局限性**：评估全部基于通用自然图像分割数据集，未涉及遥感、医学影像、文档分析等领域，通用性仍待验证。
- **基础模型依赖**：整个框架的性能上限完全受制于冻结的 SAM 3 模型，若 SAM 3 本身在某些场景下有偏，方法无法自行修正。
- **未提供极端计算环境对比**：虽展示了在 5090 上的 FPS，但未对比在更受限算力（如边缘设备）下的表现，也未分析内存占用等指标。
